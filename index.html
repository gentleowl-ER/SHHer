📋 設定步驟（管理員一次性操作）
步驟一：建立 Firebase 專案
前往 console.firebase.google.com → 點「新增專案」→ 輸入名稱（如 em-cbme）→ 停用 Analytics → 點「建立專案」
步驟二：建立 Firestore 資料庫
左側選「建構 → Firestore Database」→「建立資料庫」→ 選「正式版模式」→ 位置選「asia-east1（台灣）」→「啟用」
步驟三：開放讀寫權限
進入 Firestore →「規則」頁籤 → 將規則改為下方內容 → 點「發布」：
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
步驟四：取得設定碼
點左上角「齒輪 → 專案設定」→ 往下找「您的應用程式」→ 點網頁圖示「</>」→ 輸入名稱（如 em-web）→「註冊應用程式」→ 複製 apiKey、authDomain、projectId、appId 四個值
步驟五：填入下方欄位並連接
填入四個值 → 點「🔗 連接並寫入設定」→ 看到「✅ 連線成功」→ 點「⬆ 同步到雲端」上傳所有現有資料
其他主治醫師：打開同一個 HTML 檔案，填入相同的四個值 → 點「⬇ 從雲端同步」即可

📱 手機持續連線失敗？— 真正徹底的解法
為什麼「本機開啟」和「Google Drive 開啟」都不可靠？
本機 HTML 檔案（file://）和 Google Drive 的「預覽」功能都不是真正的網頁伺服器。Drive 預覽常以唯讀/安全模式顯示內容，內嵌的 JavaScript 經常不會被執行 —— 也就是說頁面看起來正常，但同步功能可能根本沒有啟動過。
唯一徹底的解法：把檔案放到真正會執行程式碼的網頁主機上，取得一個 https:// 開頭的正式網址。
✅ 方案一：GitHub Pages（推薦，永久免費、最穩定）
1. 前往 github.com 註冊免費帳號（已有的話略過）
2. 點右上角「+」→「New repository」→ 命名（如 em-cbme）→ 設為 Public → Create
3. 進入該 repository → 「Add file」→「Upload files」→ 把這個 HTML 檔案拖進去，檔名改為 index.html → Commit
4. 左側「Settings」→「Pages」→ Source 選「main」分支 → Save
5. 等 1-2 分鐘，會得到一個網址，格式如：https://你的帳號.github.io/em-cbme/
6. 把這個網址傳給所有人，用瀏覽器打開（不是下載檔案），手機/電腦都能穩定同步
✅ 方案二：GitHub Gist + githack（不需建立 repository，更快速）
1. 前往 gist.github.com（需登入 GitHub 帳號）
2. 檔名填 index.html，把整個 HTML 內容貼上 → 「Create public gist」
3. 點右上「Raw」按鈕，複製網址列的網址（類似 https://gist.githubusercontent.com/.../raw/.../index.html）
4. 在該網址前面加上 https://raw.githack.com/（去掉原本的 https://），組成新網址
5. 用這個新網址打開即可，會是正式執行 JS 的網頁，非常穩定
過渡方案（仍有風險，僅供緊急測試）
若暫時無法設定上述方式：用 Safari（iOS） 或 Chrome（Android） 直接開啟本機檔案，並點下方「🔍 連線診斷」確認狀態。但此方式仍可能因瀏覽器安全限制而間歇性失敗，不建議長期使用。
⚠️ 常見問題
手機持續連線失敗（最常見原因） → 此 Firebase 專案的 API Key 若設定了「HTTP 來源限制」，本機 HTML（file://）開啟時瀏覽器不會送出網址來源，會被 Google 拒絕。
　→ 解法：管理員登入 Google Cloud Console → API 和服務 → 憑證，找到對應的 API Key，將「應用程式限制」設為「無」，或改用「Google Drive 開啟」方式（見上方）完全避開此問題
安全性規則錯誤 → 回 Firebase Console 確認 Firestore 規則已點「發布」
同步後資料不見 → 「⬇ 從雲端同步」會以雲端資料覆蓋本地，建議先用「匯出/匯入」頁面做 JSON 備份
其他人看不到資料 → 確認大家用的是同一份內建設定（Project ID: shher-d2331）
免費額度 → Firestore 免費方案每日可讀取 5 萬次、寫入 2 萬次，一般科內使用完全足夠
